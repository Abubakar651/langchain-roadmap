# Phase 4: Vector Databases & Retrieval-Augmented Generation (RAG)

An AI-powered PDF Search Assistant built with LangChain, FAISS, HuggingFace Embeddings, Groq LLM, and Streamlit.

---

## What This Project Does

Upload any PDF document and ask natural language questions about its content. The app finds the most relevant sections of the PDF and generates a grounded, source-cited answer using a large language model — without hallucinating information that isn't in the document.

## How to Use the App

1. **Upload PDFs** — Click "Browse files" in the sidebar and select one or more PDF files
2. **Index PDFs** — Click the "Index PDFs" button and wait for the green success message
3. **Ask questions** — Type your question in the chat box and press Enter
4. **View sources** — Expand "View source chunks" under any answer to see exactly which page and section was used

## How It Works (RAG Pipeline)

```
PDF File
   │
   ▼
[PyPDFLoader]          ← Extracts text page by page
   │
   ▼
[RecursiveCharacterTextSplitter]   ← Splits into 1000-char overlapping chunks
   │
   ▼
[HuggingFaceEmbeddings]   ← Converts each chunk to a 384-dim vector
   │
   ▼
[FAISS Index]          ← Stores and indexes all vectors on disk
   │
   │   ◄── User asks a question
   │
   ▼
[Similarity Search]    ← Finds top-k most relevant chunks (MMR)
   │
   ▼
[Prompt Template]      ← Formats: context + question → prompt
   │
   ▼
[Groq LLM]             ← Generates a grounded answer
   │
   ▼
Answer + Source Citations
```

## Concepts Covered

### 1. Vector Databases
- Store text as high-dimensional numerical vectors
- Enable **semantic search** — find meaning, not just keywords
- `"cat"` and `"feline"` are close in vector space even though the words differ

### 2. Embeddings
- Numerical representations of text that capture meaning
- Similar text → similar vectors → small cosine distance
- We use `all-MiniLM-L6-v2` — free, fast, 384 dimensions

### 3. FAISS Operations
```python
# Create
db = FAISS.from_documents(documents, embeddings)

# Search
results = db.similarity_search(query, k=3)

# Persist
db.save_local("faiss_index")
db = FAISS.load_local("faiss_index", embeddings, allow_dangerous_deserialization=True)

# Extend
db.add_documents(new_docs)

# Use as retriever
retriever = db.as_retriever(search_type="mmr", search_kwargs={"k": 4})
```

### 4. RAG Chain (LCEL)
```python
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)
answer = rag_chain.invoke("Your question here")
```

## Retrieval Settings (Sidebar)

| Setting | Default | Description |
|---|---|---|

## Common Issues

**`python app.py` gives warnings** → Use `streamlit run app.py` instead

**`ModuleNotFoundError: langchain.text_splitter`** → Use `from langchain_text_splitters import ...`

**`groq.APIConnectionError`** → Check internet connection and that `.env` is in the `project/` folder

**Model download on first run** → `all-MiniLM-L6-v2` (~90MB) downloads once and is cached in `~/.cache/huggingface/`
